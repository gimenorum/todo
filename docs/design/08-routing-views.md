# 08. ルーティングと画面

> 要件トレース: requirements.md「技術スタック」「画面構成とナビゲーション」「設定画面」
> 状態: 実装済（Phase 0。競合ルート/解決画面は Phase 4 で実装済） ／ 実装フェーズ: 0（競合ルートは 4）

## 8.1 ルーティング方式

- **ハッシュベース**（`#/...`）。History API は使わない＝GitHub Pages で 404 にならない（要件「技術スタック」, 要件「画面構成とナビゲーション」）。
- `router/router.ts` が `hashchange` を購読し、パース結果を `State.route` に反映（[07](./07-state-and-dom.md) の単一経路）。
- 未知ルートはタスク一覧へフォールバック。

## 8.2 ルート表

| ルート | 画面 | 説明 | フェーズ |
|---|---|---|---|
| `#/` または `#/tasks` | TaskListView | TODO 一覧（主画面） | 0 |
| `#/todo/:id` | TodoEditView | 個別編集 | 0 |
| `#/settings` | SettingsView | 設定 | 0 |
| `#/todo/:id/merge` | ConflictMergeView | 競合解決（→ [10](./10-conflict-ui.md)） | 4 |

```ts
type Route =
  | { name: 'tasks' }
  | { name: 'todo'; id: Uuid }
  | { name: 'settings' }
  | { name: 'merge'; id: Uuid };
```

## 8.3 画面（トップレベルは 2 つ）

トップレベルは **タスク** と **設定** の 2 つのみ。「同期」専用タブは作らない（要件「画面構成とナビゲーション」）。

- **TaskListView**: 一覧（id キー差分更新 / [07](./07-state-and-dom.md)）、各項目に個別同期ステータス（[09](./09-status.md)）、競合項目には「同期不具合を解決する」ボタン（[10](./10-conflict-ui.md)）。
  **並び替え・絞り込み（Phase 6）**: ツールバーで並び替えキー（`DeviceSettings.sortBy` = 手動 / 期限 / 優先度 / タイトル / カテゴリ）を選択。
  どの並びも「完了は下」（`done` 第1キー）を保つ。カテゴリ＝既存タグを流用。**手動**のときのみ各行にドラッグハンドル（⠿）を出し、
  **Pointer Events**（標準 `draggable` はタッチ非対応のため不採用）で同じ完了状態のグループ内をドラッグ。確定時に前後項目の `order`
  （フラクショナルインデックス・[03 §3.2](./03-data-model.md)）の「あいだ」を `keyBetween` で算出し、通常の編集経路（→ 同期）で反映。`order` 値は recency 同期で競合化しない（[04 §4.5](./04-sync-engine.md)）。
  絞り込み（`DeviceSettings.filter`・4 軸 AND = カテゴリ/優先度/期限バケツ/タイトル検索）は折りたたみパネルで提供し、有効時は「絞り込み中」を常時表示。並び・絞り込みは端末ごと（同期しない）。
- **TodoEditView**: 全フィールド編集。画面上部に当該項目の同期ステータス（要件「ステータス表示」）。
- **SettingsView**: クラウド連携（接続/再接続）・同期設定（間隔/「手動のみ」）・データ（エクスポート3種・インポート）・（後で）言語（要件「設定画面」）。未連携時は接続導線を常設（要件「設定画面」）。
- **ConflictMergeView**: Phase 4。タスク一覧から遷移する WinMerge ライク解決画面。

## 8.4 レスポンシブナビ

ビューポート幅で切替（ブレークポイント ~768px / 要件「画面構成とナビゲーション」）。CSS メディアクエリを主とし、必要箇所のみ `matchMedia`。

| 幅 | ナビ |
|---|---|
| PC（≳768px） | 折り畳み可能な**サイドバー**。折り畳み状態は端末ごとの UI 設定としてローカル保存（`DeviceSettings.sidebarCollapsed` / [03](./03-data-model.md)） |
| モバイル（≲768px） | 下部フッターの**タブ** |

## 8.5 ナビのバッジ（要対応時のみ）

正常時は出さない。要対応のときだけ出す（要件「画面構成とナビゲーション」, 要件「ステータス表示」）。

| バッジ | 出る条件 | 付く先 |
|---|---|---|
| 未解決の競合（件数） | `State.conflicts` が非空 | タスクタブ |
| 要再接続 / 同期エラー | `global ∈ {needs-reauth, error}` | 設定タブ |

> バッジ値は `state/selectors.ts` の派生（[07 §7.4](./07-state-and-dom.md)）。未連携時はステータス系を一切出さない（[09](./09-status.md)）。

## 8.6 関連する不変条件

- ハッシュルーティングで GitHub Pages の deep link が 404 にならない（要件「画面構成とナビゲーション」）。
- ナビは ~768px でサイドバー⇄ボトムタブに切り替わる（受け入れ基準）。
