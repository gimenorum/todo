# 17. フェーズ対応表（トレーサビリティ）

> 要件トレース: requirements.md「実装フェーズ」
> 状態: 確定 ／ 実装フェーズ: —

実装フェーズ（要件「実装フェーズ」）と設計章・成果物・タグの対応。各フェーズは指定セクションのみを対象とし、特に同期エンジンとテストを先に完成させる（要件「作業の進め方」）。

> GitHub 上のリリース種別: `v0.x.y`（Phase 0–4）は**プレリリース**、`v1.0.0`（Phase 5）から**正式リリース**（タグのメジャーで自動判定。詳細は [15 §15.4](./15-build-deploy-ci.md)）。

> Phase 6 進捗: **一覧の並び替え（5 択: 手動 D&D／期限／優先度／タイトル／カテゴリ）＋絞り込み（4 軸 AND: カテゴリ/優先度/期限/タイトル）を実装**（`feature/manual-reorder` → `v1.1.0` 予定）。手動並べ替えは `order` のフラクショナルインデックス（03+08）＋ recency 同期（[04 §4.5](./04-sync-engine.md)）、入力は Pointer Events（[08 §8.3](./08-routing-views.md)）。並び・絞り込みは端末ごと（同期しない）。残り（暗号化／S3·WebDAV／GC·squash／多言語）は未着手。
