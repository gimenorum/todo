## 0.3 設計の基本原則（全章に通底）

1. **ローカルファースト**: アプリは常にローカル（IndexedDB）を読み書きする。同期は背後でローカルとリモートを双方向にやり取りする手段にすぎない。
2. **Bring Your Own Storage**: 正本はユーザーが選んだストレージ。アプリは `list/get/put/delete` だけを要求する。
3. **UI 非依存の core**: 同期エンジンは DOM・ネットワーク・IndexedDB・`idb` を一切知らない純粋関数の集合。これによりテストが UI なしに決定的に回る（→ [16](./16-testing.md)）。
4. **CAS 非依存の正しさ**: 原子的な条件付き書き込み（CAS）が無くても `list/get/put/delete` だけで正しく動く。CAS はあれば高速化に使うだけ（受け入れ基準）。
5. **決定性**: 同じ入力からは常に同じハッシュ・同じマージ結果。タイブレークは必ず全順序で行う（→ [04](./04-sync-engine.md)）。**マージコミットも入力の純関数**（deviceId 非格納）にして同時同期でも単一先端へ収束させる。
6. **黙ってデータを失わない**: 自動マージできない食い違いだけを競合として可視化し、ユーザーが解決するまで保持する（受け入れ基準）。

## 0.4 用語集

| 用語 | 定義 |
|---|---|
| スナップショット (Snapshot) | ある時点の TODO 集合。内容アドレス指定で保存される（→ [03](./03-data-model.md)）。 |
| コミット (Commit) | `{ parents, snapshot, timestamp, deviceId }`。DAG のノード。 |
| マージコミット | parents≥2 のコミット。**blob は決定的**（`deviceId` 非格納・`timestamp` は親由来 max）＝同一マージは同一ハッシュ（→ [04 §4.1](./04-sync-engine.md)）。 |
| blob / オブジェクト | 内容の SHA-256 をキーに保存される不変データ（commit・snapshot を含む）。 |
| 内容アドレス指定 (content-addressed) | キー = 内容のハッシュ。同内容＝同キー、異内容＝異キー（衝突なしに作成できる）。 |
| 先端 (tip / head) | 「どのコミットの親でもないコミット」。コミット集合から常に再導出できる。 |
| advisory HEAD | 先端のキャッシュ（ヒント）。正しさはこれに依存しない。 |
| LCA | 最近共通祖先（の極大なもの）。3-way マージの base（→ [04 §4.4](./04-sync-engine.md)）。 |
| 3-way マージ | base・left・right の 3 点から、フィールド単位で自動マージ／競合判定する。 |
| tombstone | 削除済みを表す印（`deleted: true`）。物理削除せず履歴に残す。 |
| materialize | コミット/blob から計算した「表示用 TODO リスト」。IndexedDB に永続化する。 |
| フラクショナルインデックス | 手動並べ替え用の順序キー（`order`）。v1 は未使用の予約。 |
| PKCE | OAuth public client 向けの認可コード横取り対策（Dropbox 連携で使用）。 |
| appDataFolder | Google Drive の最小権限アプリ専用領域。 |
| BYOS | Bring Your Own Storage。 |

## 0.5 受け入れ基準と章の対応（不変条件のインデックス）

各受け入れ基準（要件の同名セクション）を、それを保証する章へ紐付ける。詳細は各章の「不変条件」節に再掲する。

| 受け入れ基準 | 保証する主な章 |
|---|---|
| すべての読み書きがオフラインで動作 | 06, 07 |
| 別項目/片側変更は自動両立、同一フィールド食い違いのみ競合・非消失 | 04, 10 |
| 正しさが原子的 CAS に非依存 | 04, 05 |
| 読み込み時に再ハッシュ検証 | 04 |
| リスト変更は id キー差分更新・フォーカス/スクロール維持 | 07 |
| 未連携時は同期系 UI を出さない | 09 |
| 同期設定は端末ごと・同期データに載らない／単独で入出力 | 06, 11, 13 |
| ナビは ~768px でサイドバー⇄ボトムタブ | 08 |
| iOS は前面復帰/起動時に同期・ローカル消失から復旧 | 06, 11, 12 |
| JSON エクスポート→インポートが無損失往復・マージ取り込み | 13 |
